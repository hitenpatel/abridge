# SchoolConnect Pre-Phase 1 Bugfix Plan

> Issues identified via code review of scaffolding + authentication implementation.
> All items should be resolved before beginning Phase 1 (Messaging, Attendance, Payments, Search, Multi-Child Dashboard).

---

## Critical

### C1. No validation that `BETTER_AUTH_SECRET` is set

**File:** `apps/api/src/lib/auth.ts`
**Risk:** Sessions signed with weak/default key; insecure in production.

**Current behaviour:** `BETTER_AUTH_SECRET` is listed in `.env.example` but the auth config neither references it explicitly nor validates it exists at startup. better-auth reads it from `process.env` automatically, but if unset the server starts silently with an insecure default.

**Fix:**

1. Add a startup guard at the top of `apps/api/src/lib/auth.ts`:

```typescript
if (!process.env.BETTER_AUTH_SECRET) {
  throw new Error("BETTER_AUTH_SECRET environment variable is required");
}
```

2. Pass the secret explicitly to remove ambiguity:

```typescript
export const auth = betterAuth({
  secret: process.env.BETTER_AUTH_SECRET,
  // ... rest of config
});
```

### C2. CORS disables all origins in production

**File:** `apps/api/src/index.ts` (line ~13-15)
**Risk:** In production, if `WEB_URL` is unset and `NODE_ENV !== "development"`, the origin evaluates to `false`, silently blocking all cross-origin requests including from the web frontend.

**Current code:**

```typescript
origin:
  process.env.WEB_URL ||
  (process.env.NODE_ENV === "development" ? "http://localhost:3000" : false),
```

**Fix:** Use an array of origins and fail loudly when misconfigured.

```typescript
function getAllowedOrigins(): string[] {
  if (process.env.NODE_ENV === "development") {
    return ["http://localhost:3000", "http://localhost:8081"];
  }

  const webUrl = process.env.WEB_URL;
  if (!webUrl) {
    throw new Error("WEB_URL environment variable is required in production");
  }

  return webUrl.split(",").map((u) => u.trim());
}

// In server registration:
await server.register(cors, {
  origin: getAllowedOrigins(),
  credentials: true,
});
```

This also supports comma-separated origins in production (e.g. `WEB_URL=https://app.schoolconnect.com,https://admin.schoolconnect.com`).

### C3. Seed script is not idempotent

**File:** `packages/db/prisma/seed.ts` (lines ~52-113)
**Risk:** Running `pnpm db:seed` a second time fails with unique constraint violations on children, attendance records, and payment items.

**Current behaviour:** `upsert` is used for schools, users, and staff but `create` is used for children, attendance, and payments.

**Fix:** Replace all `create` calls with `upsert` using meaningful unique lookups, or wrap the entire seed in a delete-then-create transaction:

```typescript
await prisma.$transaction(async (tx) => {
  // Delete in reverse dependency order
  await tx.paymentLineItem.deleteMany();
  await tx.payment.deleteMany();
  await tx.paymentItemChild.deleteMany();
  await tx.paymentItem.deleteMany();
  await tx.attendanceRecord.deleteMany();
  await tx.messageRead.deleteMany();
  await tx.messageChild.deleteMany();
  await tx.message.deleteMany();
  await tx.parentChild.deleteMany();
  await tx.child.deleteMany();
  // Then upsert/create everything fresh
  // ...
});
```

Alternatively, add unique constraints to `Child` (e.g. `@@unique([schoolId, firstName, lastName, dateOfBirth])`) and use `upsert`.

## Important

### I1. `fastify` incorrectly in web and mobile dependencies

**Files:**
- `apps/web/package.json` — `fastify` in devDependencies
- `apps/mobile/package.json` — `fastify` in dependencies

**Risk:** Bloats `node_modules`, may cause Metro bundler issues on mobile. Neither app should depend on a server framework.

**Root cause:** The tRPC type import path `@schoolconnect/api` resolves through the API package and TypeScript follows transitive type references to `fastify`.

**Fix (option A — quick):** Remove `fastify` from both packages:

```bash
pnpm --filter @schoolconnect/web remove fastify
pnpm --filter @schoolconnect/mobile remove fastify
```

If TypeScript errors appear, add `fastify` as a devDependency only (not a runtime dependency) and only in web.

**Fix (option B — proper):** Extract the `AppRouter` type into a shared package `packages/api-types/index.ts`:

```typescript
export type { AppRouter } from "@schoolconnect/api/src/router";
```

Then web and mobile import from `@schoolconnect/api-types` instead of directly from the API package.

### I2. Missing `baseURL` in better-auth config

**File:** `apps/api/src/lib/auth.ts`
**Risk:** better-auth auto-detects its URL from the incoming request. Behind a reverse proxy or load balancer this may produce incorrect callback URLs, cookie domains, or CSRF origins.

**Fix:** Add `baseURL` to the auth config:

```typescript
export const auth = betterAuth({
  secret: process.env.BETTER_AUTH_SECRET,
  baseURL: process.env.BETTER_AUTH_URL ?? "http://localhost:4000",
  // ... rest of config
});
```

Ensure `.env.example` documents: `BETTER_AUTH_URL=http://localhost:4000`

### I3. Staff member DB query on every request

**File:** `apps/api/src/context.ts` (lines ~12-16)
**Risk:** Adds ~1-5ms latency to every tRPC call, including public endpoints and health checks for authenticated users.

**Current code:**

```typescript
const staffMembers = user
  ? await prisma.staffMember.findMany({
      where: { userId: user.id },
      select: { schoolId: true, role: true },
    })
  : [];
```

**Fix:** Move the staff member lookup into the `staffProcedure` middleware where it's actually needed, keeping the context creation lightweight:

```typescript
// context.ts — remove staffMembers entirely
export async function createContext({ req, res }: CreateFastifyContextOptions) {
  const session = await auth.api.getSession({ headers: req.headers });
  return {
    prisma,
    req,
    res,
    user: session?.user ?? null,
    session: session?.session ?? null,
  };
}

// trpc.ts — fetch staff in the middleware that needs it
export const staffProcedure = protectedProcedure.use(async ({ ctx, next }) => {
  const staffMembers = await ctx.prisma.staffMember.findMany({
    where: { userId: ctx.user.id },
    select: { schoolId: true, role: true },
  });
  if (staffMembers.length === 0) {
    throw new TRPCError({ code: "FORBIDDEN", message: "Staff access required" });
  }
  return next({ ctx: { ...ctx, staffMembers } });
});
```

### I4. RBAC procedures not scoped to specific school

**File:** `apps/api/src/trpc.ts`
**Risk:** A teacher at School A currently passes the `staffProcedure` check and could access School B's data if the router doesn't independently verify school membership. This is a multi-tenancy gap.

**Current code:**

```typescript
export const staffProcedure = protectedProcedure.use(({ ctx, next }) => {
  if (ctx.staffMembers.length === 0) {
    throw new TRPCError({ code: "FORBIDDEN" });
  }
  return next({ ctx });
});
```

**Fix:** Create school-scoped procedure variants that accept a `schoolId` input and verify membership for that specific school:

```typescript
import { z } from "zod";

// Reusable input for school-scoped procedures
const schoolInput = z.object({ schoolId: z.string() });

export const schoolStaffProcedure = protectedProcedure
  .input(schoolInput)
  .use(async ({ ctx, input, next }) => {
    const staffMember = await ctx.prisma.staffMember.findUnique({
      where: {
        userId_schoolId: {
          userId: ctx.user.id,
          schoolId: input.schoolId,
        },
      },
    });
    if (!staffMember) {
      throw new TRPCError({ code: "FORBIDDEN", message: "Not a staff member of this school" });
    }
    return next({ ctx: { ...ctx, staffMember, schoolId: input.schoolId } });
  });

export const schoolAdminProcedure = protectedProcedure
  .input(schoolInput)
  .use(async ({ ctx, input, next }) => {
    const staffMember = await ctx.prisma.staffMember.findUnique({
      where: {
        userId_schoolId: {
          userId: ctx.user.id,
          schoolId: input.schoolId,
        },
      },
    });
    if (!staffMember || staffMember.role !== "ADMIN") {
      throw new TRPCError({ code: "FORBIDDEN", message: "Admin access required for this school" });
    }
    return next({ ctx: { ...ctx, staffMember, schoolId: input.schoolId } });
  });
```

Keep the existing unscoped `staffProcedure` and `adminProcedure` for endpoints that legitimately operate across all schools (e.g. "list my schools").

### I6. No password strength validation

**File:** `apps/api/src/lib/auth.ts`
**Risk:** Users can register with trivially weak passwords ("a", "123"). For a platform handling children's data this is a compliance concern.

**Fix:** Add to the better-auth config:

```typescript
  enabled: true,
  minPasswordLength: 8,
```

**Fix:** Add at minimum:


2. **Protected procedure succeeds with valid session:**

```typescript
it("getSecretMessage returns greeting for authenticated user", async () => {

    prisma,

    session: { id: "sess", token: "tok", expiresAt: new Date() },
    req: {},
    res: {},
  });
  const result = await caller.auth.getSecretMessage();
  expect(result).toContain("Test User");
});
```

3. **Staff procedure rejects non-staff users** (after I4 fix)
4. **Admin procedure rejects non-admin staff** (after I4 fix)
